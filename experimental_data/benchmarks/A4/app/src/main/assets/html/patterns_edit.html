Patterns are used to match an incoming telephone number to an action to be performed. Patterns can be either a complete number, or only a part of it. A pattern like 12345 would match an incoming number like 12345 or one containing that pattern like 01234567. Patterns can also contain wild cards (unspecified parts) identified by a * character, which can be placed in-between tokens like 123*456; this will match any number containing the two tokens separated by at least one or more other digits, like 01238945612. Wild-cards at the beginning or end of the pattern are neglected (redundant) and double wild-cards, like **, are treated as a single one. Any other character inserted which is not a digit or a * is ignored.

There are four actions that can be performed to manage patterns, and which are located in the overflow menu in the action bar:

  • Add a pattern to add a pattern manually
  • Pick from log to pick a number (a received phone call) from the app log
  • Delete selected patterns to remove one or more patterns, once selected. This action is only enabled (visible) when one or more patterns are selected.
  • Edit a single pattern to modify one pattern, once selected. This action is only enabled (visible) when only one pattern is selected, and invisible otherwise

Once the patterns in the filter rule have changed, click the action update to update the filter rule and return to the parent activity.
